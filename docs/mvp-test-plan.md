# 🧪 MVP 測試計畫 (Minimum Viable Product Test Plan)

## 1. 測試時程與目標
| 階段 | 週期 | 目標 |
| :--- | :--- | :--- |
| **第一階段：單次測試** | 1 次 | 驗證全流程（會不會跑）、報表輸出與資料庫路徑。 |
| **第二階段：穩定性測試** | 3 個交易日 | 驗證連續執行穩定性（穩不穩）、API 邊界條件與格式變動。 |
| **第三階段：準確性觀察** | 2 週交易日 | 驗證 T+5/T+20 績效參考價值（準不準）與優化建議。 |

### 批次驗證解讀原則
- **1-2 日**：僅能視為 smoke test，確認流程與閉環正常。
- **3-5 日**：可視為 stability test，適合檢查 action 語意與連續執行穩定性。
- **6-10 日**：可做 early observation，開始比較 action / rule / thesis 的 Alpha 表現。
- **10 日以下的 insights**：只可作 early-stage tuning，不應直接當成砍規則或升權依據。

## 2. MVP 測試樣本名單 (混合樣本)
| 類別 | 代號 | 名稱 | 測試用途 |
| :--- | :--- | :--- | :--- |
| **穩定權值股** | `2330` | 台積電 | 驗證基本面/財報資料完整性，作為研究基準。 |
| **題材強勢股** | `3363` | 上詮 | 驗證新聞加成、量價過熱與 eventScore 敏感度。 |
| **潛力成長股** | `1560` | 中砂 | 驗證營收動能、論點 (Thesis) 狀態判斷。 |
| **潛力成長股** | `6187` | 萬潤 | 驗證規則在「非純題材」成長股上的判斷力。 |
| **弱勢風險股** | `2002` | 中鋼 | 驗證風險規則、SELL/EXIT 邏輯與防呆機制。 |

## 3. 測試執行指令
### A. 指定股票研究 (Phase 1/2)
```bash
# 研究指定 5 檔樣本
npm run candidates -- --stocks=2330,3363,1560,6187,2002
```

### B. 成效與分析
```bash
npm run run-history latest
npm run outcomes latest
npm run performance latest
npm run insights latest
```

## 4. 關鍵檢查點 (Checkpoints)
- [ ] **流程面**: 指令是否能無人值守跑完？是否有卡住或 Uncaught Exception？
- [ ] **資料面**: 2002 (中鋼) 是否常缺新聞？3363 (上詮) 的 eventScore 是否合理？
- [ ] **決策面**: 2330 (台積電) 決策是否過於頻繁變動？
- [ ] **洞察面**: 系統是否能指出哪條規則最常導致 3363 的誤判？
